Datenanalyse mit R

Warum tidyverse?

  • Konsistenz
    • z.B. alle stringr Funktionen nehmen einen string als erstes Argument
    • z.B. die meisten Funktionen nehmen den data frame als erstes Argument (piping)
  • Es zwingt zu einer sorgfältigen Vorgehensweise
  • Synergien zwischen verschiedenen Paktenen/tools
  • Implementiert einfache Lösungen zu allgemeinenen Problemen
  • Smartere default Einfstellung
    • z.B. utils::write.csv(row.names = FALSE), readr::write_csv()
  • Funktionen sind oft wesentlich schneller (viele sind mit Rcpp umgesetzt)
  • Zunahme an Paketen die das tidyverse Konzept implementieren

Paket laden (JEDE SESSION)

Ein Paket muss für jede session geladen werden um auf die Funktionen zugreifen zu können. Dazu gibt es library() und require(). R überprüft dann erstmal, ob dieses Paket auch installiert. Der Unterschied zwischen beiden Funktionen ist nur was passiert, wenn das Paket nicht vorhanden ist:

library(any_package) # library("any_package") would also work

## Error in library(any_package): there is no package called 'any_package'

require(any_package) # require("any_package")
Datenanalyse mit R

Paket laden (JEDE SESSION)

Wenn ein spezifisches Paket geladen wird, wird es als default zum Suchpfad hinzugefügt:

modified from Advanced R by H. Wickam, 2014

  • Um eine Funktion aufrufen zu können, muss R sie erstmal finden. Dafür sucht als erstes in der globalen Umgebung.
  • Anschl. sucht R im Suchfad, also der Liste aller geladenen Pakete, in einer spezifischen Reihenfolge.
  • Wenn R mehrere Funktionen mit dem gleichen Namen findet, verwendet es die Funktion des zuletzt geladenen Pakets.

Import optimieren - Zeilen auslassen

Die ersten n Zeilen mit Metadaten am Anfang einer Datei können ausgelassen werden mit skip = n:

read_csv("The first line of metadata
  The second line of metadata
  x,y,z
  1,2,3", skip = 2)

## # A tibble: 1 x 3
##       x     y     z
##   <dbl> <dbl> <dbl>
## 1     1     2     3
Datenanalyse mit R

Import optimieren - Spaltennamen

Wenn die Datei keine Spaltennamen enthölt: col_names = FALSE; R vergibt dann die Labels X1 bis Xn:

read_csv("1,2,3
  4,5,6", col_names = FALSE)

## # A tibble: 2 x 3
##      X1    X2    X3
##   <dbl> <dbl> <dbl>
## 1     1     2     3
## 2     4     5     6

Du kannst auch einen gesonderten character vector als col_names übergeben:

read_csv("1,2,3
  4,5,6", col_names = c("x", "y", "z"))

## # A tibble: 2 x 3
##       x     y     z
##   <dbl> <dbl> <dbl>
## 1     1     2     3
## 2     4     5     6

Was sind eigentlich tibbles?

  • Das 'tibble' Pakets ermöglicht eine neue 'tbl_df' Klasse (das 'tibble') striktere checks bereit stellt und sich besser formatieren lässt als die traditionellen data frames.
  • Alle Funktionen für data frames funktionieren auch bei tibbles
  • Alle Pakete generieren tibbles automatisch
  • Mehr zu tibbles gibt es hier: vignette("tibble").
Datenanalyse mit R

Was sind eigentlich tibbles?

  • Functions for data frames will work also for tibbles.
  • All tidyverse packages generate tibbles automatically
  • To learn more check vignette("tibble").

Daten speicher und exportieren

Daten als R Objekte speichern

Du kannst Dein tibble oder data frame als .R Objekt abspeichern und jederzeit später wieder laden mi save(your_tibble, "filename") und load("filename"):

save(hydro, file = "Mein_erstes_Objekt.R")
# Wir löschen hydro aus der globalen Umgebung
rm(hydro)
hydro
load(file = "Mein_erstes_Objekt.R")
hydro # jetzt sollte es wieder da sein
Datenanalyse mit R

Exportieren von Daten

  • Wenn Du Deine Daten expoertieren willst um sie dann mit anderen Programmen zu bearbeiten verwende am besten das gleiche Format wie beim Import.
  • Die meisten Importfunktionen haben ein Äquivalent zum Exportieren:
    • read_delim() --> write_delim()
    • read_csv() --> write_csv()
    • read_tsv() --> write_tsv()
    • other functions: write_excel_csv()
write_csv(hydro, "Mein_hydro_Datensatz.csv")
